# Mod Analysis Tasks

## Phase 1 - Structure
- Summarize the folder structure
- Identify main mod files
- Identify config/data/code files
- Identify dependencies on BepInEx, Harmony, or FFU

## Phase 2 - File-by-file explanation
For each significant file:
- Explain what it likely does
- Explain how it affects gameplay
- Explain key fields, methods, or config entries
- Note any likely risks or unknowns

## Phase 3 - Comment rewrite
- Replace unclear comments with plain English comments
- Add missing comments where behavior is non-obvious
- Do not add excessive commentary

## Phase 4 - Modder summary
Produce:
- "How this mod works"
- "How to modify it safely"
- "What to test in game"
